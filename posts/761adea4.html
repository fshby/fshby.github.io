使用注册表添Windows桌面右键菜单快捷键

查看教程

说明：当使用方法1时,可以右键点击桌面菜单打开；使用方法2，可以右键点击文件夹打开菜单
方法1：
参考链接

  1. 使用快捷键windows+R运行regedit打开注册表
  2. 找到HKEY_CLASSES_ROOT/Directory/Background/shell 右键新建项 更改默认右键菜单名
  3. 打开新建的项，新建command文件夹中更改默认源程序路径
    例：E:\小程序\Microsoft VS Code\Code.exe

    例：E:\小程序\Microsoft VS Code\Code.exe

  1. 使用快捷键windows+R运行regedit打开注册表

system进程占用80或443端口

  1. 查看端口占用进程 netstat -ano |findstr 80
  2. 确认是system进程占用端口无法终止
  3. 运行regedit打开注册表，\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\HTTP，降start值3（初始值）改为4（ok）或0（nok）

U盘问题

没办法识别其他人的U盘，自己的U盘正常使用

产看解决方案

当电脑无法识别其他人的U盘时，可能涉及硬件、软件或系统设置等多方面原因。以下是详细的排查步骤和解决方案：

一、初步检查

  1. 确认U盘物理连接

    • 重新插拔U盘：确保U盘完全插入USB接口，避免接触不良。
    • 更换USB接口：尝试将U盘插入电脑的其他USB端口（尤其是主板背面的接口），排除接口故障。
    • 检查U盘状态：观察U盘指示灯是否亮起，或尝试在其他设备（如手机、其他电脑）上测试U盘是否正常工作。

  2. 检查设备管理器

    • 按 Win + X 选择 设备管理器，展开 通用串行总线控制器。
    • 如果看到黄色感叹号或红色叉号，可能是驱动问题。右键点击相关设备，选择 更新驱动程序 或 卸载设备 后重新插入U盘。

二、软件与系统设置

  1. 启用USB设备

    • 电源管理设置：
      • 在设备管理器中，右键点击 USB根集线器，选择 属性 > 电源管理。
      • 取消勾选 “允许计算机关闭此设备以节约电源” 。
    • BIOS设置：
      • 重启电脑，进入BIOS（通常按 Del、F2 或 F10）。
      • 检查 USB Legacy Support 或 USB 2.0/3.0 Controller 是否启用。

  2. 禁用安全软件干扰

    • 暂时关闭杀毒软件、防火墙或第三方安全工具（如360、卡巴斯基），某些软件可能阻止外部设备识别。

  3. 检查磁盘管理

    • 按 Win + R 输入 diskmgmt.msc 打开磁盘管理。
    • 查看U盘是否显示为未分配空间或未格式化。如果是，可右键选择 新建简单卷 并格式化。

—

  • 重置USB设置：
    在注册表编辑器（regedit）中导航至 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\USBSTOR，将 Start 值改为 3（启用）。

—
总结与建议
| 问题类型 | 解决方案 |
|————————|———————————————————|
| 物理连接问题 | 更换USB接口，检查U盘状态 |
| 驱动问题 | 更新/重装USB驱动，检查BIOS设置 |
| 系统设置冲突 | 禁用电源管理，关闭安全软件 |
| 硬件故障 | 测试其他U盘，检查电脑USB端口 |
| 病毒/恶意软件 | 全盘扫描U盘，使用杀毒软件 |

英伟达驱动跟新灾难

RTX50驱动翻车实录：
1月更新后黑屏死机高频发,修复补丁导致GPU温度显示异常,G-Sync适配出现迷之卡顿

  • 重点问题汇总:
    1) 0576.02驱动温度监测不准
    2) 最新576.15仍存在游戏崩溃
    3) 官方确认至少15项漏洞待修复

  • 建议操作:

1) 暂停驱动自动更新
2) 暂时回退551.23稳定版
3) 关闭Windows自动更新

  • 解决办法：
    方案一： 开机按F2进入bios使用启动盘回退显卡驱动到稳定版本
    方案二： 开机按F7进去安全模式卸载最新更新驱动，重启后打开应该可以正常显示，打开设备管理器查看显示适配器中
    Intel(R) UHD Graphics 630与NVIDIA GeForce GTX 1050 Ti 两个驱动无异常，基本就没什么问题了
