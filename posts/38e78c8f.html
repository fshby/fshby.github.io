探针测试

发表于2023-07-30|更新于2023-08-08|工作日志
|阅读量:

播放器事件测试

安装拉起软件探针

1.通过 U 盘启动播放器事件测试工具（VideoBroadCastTest.apk,包名为
“com.chinamobile.middleware.videotest.test”，需列入平台配置的白名
单）

  1. 安装探针软件：pm install VideoBroadCastTest.apk
  2. 拉起软件探针：am start -n com.chinamobile.middleware.videotest.test/com.chinamobile.middleware.videotest.test.MainActivity
  3. 查看so进程和apk的进程是否正常运行
  4. 在播放器事件测试工具的界面中点击按钮“开始播放器事件测试”后，进行

打开视频

  1. 先显示PLAY_PREPARE
  2. 视频出现第一帧画面后显示PLAY_START字段
  3. 两个字段之间不允许出现BUFFER_START和BUFFER_END字样

seek操作（快进、快退）

  1. SEEK_START与SEEK_END字段成对出现
  2. 两个字段之间不允许出现BUFFER_START和BUFFER_END字样

暂停操作

  1. 暂停后显示PAUSE_MESSAGE
  2. 恢复播放后显示RESUME_MESSAGE字段或者包含“SEEK_START”和“SEEK_END ”字样

返回操作

“回看”节目不支持自动播放完毕，该条测试手动退出视频播放

  1. 返回退出播放或者视频正常播放完毕显示PLAY_QUIT

限速卡顿

  1. BUFFER_START和BUFFER_END字段成对出现
  2. 视频出现卡顿时，屏幕右侧显示“PLAYABE_REPORT”字样同一行内的对应可播时长
    的值为 0-3 秒

剩余可播放时长

  1. PLAY_PREPARE的系统时间和第一个PLAYABE_REPORT时间间隔为10s左右（允许500ms误差）
  2. PLAYABE_REPORT字段每隔10s弹出一次（允许500ms误差）

ID属性测试

  1. 执行播放器操作事件，视频的ID后的紫色数字应保持不变
  2. 播放相同或不同的视频。视频播放ID的属性值不一样

时长准确性测试

  • 首次加载时长准确性
  1. 开启秒表功能，点击视频录制按钮，打开任意视频，
  2. 记录出现黑屏的第一帧与视频播放的第一帧对应的秒表时间，
  3. 与“PLAY_START”内的 costtime 的值播放器上报的首次加载时长校对（误差小于1s）
  • 卡顿时长准确性
  1. 开启秒表功能，点击视频录制按钮，打开任意视频，
  2. 记录视频卡顿的第一帧与视频结束卡顿的第一帧对应的秒表时间，
  3. 与“Buffer_End”的costtime”的值为播放器上报的卡顿时长校对（误差小于500ms）
  • 播放时长准确性
  1. SEEK_START中playtime的值与播放时长一致（误差小于1s）
  2. “PAUSE_MESSAGE” 中 的playtime 的值播放进度条播放的播放时长一致（误差小于1s）
  3. “BUFFER_START”内的 playtime 的值与播放进度条播放的播放时长一致（误差小于1s）
  4. 视频退出播放时的进度条与“PALY_QUIT” 里 的playtime 的值一致（误差小于1s）
  5. 重复播放，“PLAY_START”或“SEEK_START”字样的后面跟着的 playtime 的时间值与上一次退出的已播放时长一致（误差小于1s）

显示尺寸大小

  1. 使用crt命令实时查看窗口大小
  2. 与Display WxH的值进行比较
    i

软探针版本效验

  1. 过滤 http contains boot，查看boot 中的版本信息
  2. boot 信息中“version”所显示软探针 APK 版本及libpcapcmcc.so 版本与需预置的组件版本一致
  3. 同一版本的 APK 版本信息（如右例中“18230”）和libpcapcmcc.so 的版本信息（如右例中“18234”）差值的绝对值不超过 50

收视数据效验

上报数据

  • 注：
  1. 第一条为播放1min后上报的节目，点击该条->右键->追踪流->tcp流，查看”subEvent”参数， “subEvent”:61
  2. 第二条为影片播放11min后上报的节目，点击该条->右键->追踪流->tcp流，查看”subEvent”参数， “subEvent”:62
  3. 第三条为影片播放11min后按遥控器返回键退出播放后上报的节目，点击该条->右键->追踪流->tcp流，查看”subEvent”参数， “subEvent”:63
  4. 61,62,63全部都抓到证明节目正常上报

平均下载度

  1. 过滤 http contains programInfo，查看显示平均下载速度字段
  2. 平均下载速度，avgDownloadThroughput 的值不为0

IPTV告警数据触发

1.制造花屏和欠载方法：拔掉路由器W口
2.开机后抓包，播放影片后制造花屏或欠载如12s，欠载或花屏结束（UNLOAD_END/BLURREDSCREEN_END）2min后告警数据上报，打开抓包文件，输入过滤条件http contains alarm

对接工作

发邮件到，对接负责人后，对方会发一个已经签过字的pdf文档，这边我们签过字后再回传给对方即可

性能方面

  1. 拷机“点播影片”正常播放12小时以上，终端性能收视数据正常上报
  2. 拷机“直播节目”正常播放12小时以上，终端性能收视数据正常上报
  3. “点播节目”播放12小时以上，
    1) 应用端的CPU不呈持续上升状况 ，cpu均值不超过系统cpu的5%；应用端的内存不呈持续上升状况 ；
    2) 内存均值不超过系统内存的 5%
  4. “直播节目”播放12小时以上，
    1) 应用端com.cmcc.mid.softdetector 的CPU 不呈持续上升状况 ，cpu均值不超过系统 cpu的5%
    2) 应用端的内存不呈持续上升状况
