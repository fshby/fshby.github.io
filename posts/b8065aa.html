贝科姆

USB→TTL —CH340G
贝科姆是一个基于CH340G USB转串口芯片的单片机线刷机版。其TTL接口包括5V、3.3V、TXD、RXD和GND。
通过使用USB-TTL转换模块（如CH340G）将计算机上的USB接口与单片机连接，可以实现从计算机向单片机上传代码或固件等功能。

请确保在操作时仔细阅读并遵守相关的硬件规范和使用说明，以避免损坏设备或造成其他不良后果。

  • TTL接口5v、3V3、TXD、RXD、GND

    5V：提供5伏电压，可用于给外部设备供电。
    3V3：提供3.3伏电压，可用于给特定的外部设备供电。
    TXD：传输引脚，用于发送数据。
    RXD：接收引脚，用于接收数据。
    GND：地线引脚，用于电路的接地。

  • 注意：靠近外侧的第二孔为地线

  • rstack数据包
    用于强制关闭TCP链接。TCP连接关闭的正常方法是四次握手。但四次握手不是关闭TCP连接的唯一方法有时，如果主机需要尽快关闭连接或连接超时，端口或主机不可达，RSTReset包将被发送注意，由于RST包不是TCP连接中的必须部分，可以只发送RST包即不带ACK标记，但在正常的TCP连接中RST包可以带ACK确认标记。

打开wireshark循环条弹窗提示：The helper program for “Admin-only Mode

错误原因：在第一次安装，wireshark时，安装nmap时勾选了多余选项Restrict Npcap driver's access to Administrators only—
仅限制Npcap驱动程序访问管理员
解决办法：卸载Nmap软件，重新安装取消勾选此选项

  • 过滤方式
    Wireshark提供多种报文过滤方式，用于筛选和查找特定的网络数据包。以下是一些常见的Wireshark报文过滤方式：
  1. IP地址过滤：使用过滤器”ip.addr”或”ip.src”和”ip.dst”可以过滤指定源IP地址 或目标IP地址的数据包。
  2. 端口过滤：使用过滤器”tcp.port”或”udp.port”可以过滤指定TCP或UDP端口的数据 包。
  3. 协议过滤：使用过滤器”tcp”、”udp”、”icmp”等可以过滤特定的传输层协议数据包。
  4. 协议字段过滤：使用过滤器”http”、”dns”、”ftp”等可以过滤包含特定协议字段的数 据包。
  5. 数据包长度过滤：使用过滤器”frame.len”可以过滤指定数据包长度范围内的数据包。
  6. 时间过滤：使用过滤器”time”或”timeframe”可以过滤指定时间范围内捕获的数据 包。
  7. MAC地址过滤：使用过滤器"ether host mac"、”eth.addr mac”或”eth.src mac”和”eth.dst mac”可以过滤指定源 MAC地址或目标MAC地址的数据包。
  8. ICMP类型过滤：使用过滤器”icmp.type”可以过滤指定类型的ICMP数据包。
  9. 表达式组合：可以结合使用多个过滤器，通过逻辑运算符如”and”、”or”、”not”等来 组合过滤条件，实现更复杂的过滤操作。
  • http过滤方式
    在Wireshark中，HTTP协议过滤字符串的用法是通过使用过滤器来筛选并查找HTTP协议相关的数据包。HTTP协议是一种用于在Web浏览器和服务器之间传输数据的协议，通过对HTTP协议过滤字符串进行过滤，可以找到特定的HTTP请求或响应数据包。
    使用Wireshark进行HTTP协议过滤字符串时，可以使用以下过滤器：
  1. 过滤HTTP请求：使用过滤器http.request可以仅显示HTTP请求数据包。例如
    http.request.method == “GET”，这将只显示HTTP的GET请求数据包。
  2. 过滤HTTP响应：使用过滤器http.response可以仅显示HTTP响应数据包。例如：http.response.code == 200，这将只显示状态码为200的HTTP响应数据包。
  3. 过滤指定URL：使用过滤器http.host可以筛选指定的URL。例如：http.host == “www.example.com”，这将只显示与”www.example.com”相关的HTTP数据包。
  4. 过滤指定关键字：使用过滤器http contains可以找到包含指定关键字的HTTP数据包。例如：http contains “login”，这将显示包含”login”关键字的HTTP数据包。
  5. 组合过滤：可以组合多个过滤条件，如http.request.method == “POST” && http contains “password”，这将找到使用POST方法提交密码的HTTP请求数据包。

内置探针是指在计算机网络中的一种技术工具或设备，用于监测和评估网络的性能、可用性和健康状况。它通常被部署在网络设备、服务器、路由器或其他相关设备内部，并持续地进行可编程的网络测试和检测。

  • 内置探针可以提供以下功能和特点：
    1. 网络性能监测：内置探针可以定期测量网络连接的带宽、延迟、丢包率等性能指标，以提供对网络质量的实时监测和评估。
    2. 故障诊断与故障排除：通过内置探针，网络管理员可以快速识别网络中的故障点和问题，并采取相应的纠正措施，以减少网络中断和提高服务可用性。
    3. 流量分析与优化：内置探针可以收集和分析网络流量数据，帮助管理员识别流量模式、瓶颈和优化潜力，从而改善网络性能和效率。
    4. 安全监测与入侵检测：内置探针可以检测和识别网络中的安全威胁、攻击行为和异常流量，提供实时的入侵检测和响应能力。
    5. 自动化管理与配置：内置探针支持自动配置和管理功能，可以根据设定的规则和策略，自动进行网络测试、警报通知和性能优化，提高网络运维的效率和可靠性。
    6. 内置探针在企业网络、数据中心、云计算环境等场景中广泛使用，以帮助管理员监测和管理复杂的网络基础设施，并确保网络的高性能、安全和稳定运行。

开机之后，启动盒子本身安卓9的系统，加载后续的网络连接，业务接入。开机后获取联网方式去连接网络，连接成功之后，再进行业务认证。
业务认证又有机顶盒支持通过配置的IPTV 业务账号、密码向 IPTV 平台发起IPTV业务认证，认证过程包括发起请求、token响应、加密上传请求、认证通过响应四个消息

  1. 发起请求:机顶盒向 IPTV 业冬平台发出 IPTV 业务认证请求;
  2. token响应:平台根据时间戳生成并下发token
  3. 加密上传请求:机顶盒将IPTV 业务账号、密码、IP 地址和token加密，并上传认证信息;
  4. 认证通过:平台校验机顶盒上传的认证信息，返回成功或出错信息到机顶盒。
    业务认证通过后，机顶盒再与NTP服务器同步。与NTP服务器同步可以确保设备播放电视节目时的时间显示准确无误，保证直播等信息的时效性

四色建

四色键（Four Color Keys）是指在遥控器或电视机上的四个特定颜色的按键，通常标注为红、绿、黄、蓝四种颜色。这些按键用于在某些电视、机顶盒或其他设备上进行快速访问和操作特定的功能或菜单。
四色键在不同设备上的具体功能可能会有所不同，但通常它们被设计用于以下常见用途：

  1. 红色键（Red Key）：通常用于与互动内容或交互功能相关的操作。
  2. 绿色键（Green Key）：通常用于与导航或选择相关的操作。
  3. 黄色键（Yellow Key）：通常用于与多媒体播放相关的操作。
  4. 蓝色键（Blue Key）：通常用于与设置相关的操作。
    需要注意的是，四色键的具体功能和用途在不同的设备和应用程序中可能会有所差异，用户应根据具体设备的说明或界面上的提示来正确使用和操作四色键。

根目录下upgrade文件夹(放置update.zip升级包和配置文件upgrade)
